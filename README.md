📌 项目简介
一款基于大模型的 App 用户评论自动分析工具。上传 CSV 文件，自动提取评论、用户名、评分，调用 DeepSeek API 进行情感分析，输出统计结果、紧急问题排序和可视化图表。

核心价值：将产品经理每周 5 小时的评论筛选时间降低到 1 小时。

🎯 解决的问题
痛点	说明
效率低	产品经理每天要手动阅读大量用户评论，耗时费力
不全面	人工阅读容易遗漏关键信息，尤其是低频但严重的问题
难量化	没有数据支撑，无法准确判断哪些问题最紧急
反馈慢	从评论到决策的链路太长，问题发现周期长
✨ 核心功能
功能	说明
自动提取	自动识别 CSV 中的评论内容、用户名、评分
情感分析	调用 DeepSeek API，判断每条评论的情感倾向（正面/负面/中性）
问题提取	自动总结每条评论的核心问题，一句话概括
紧急度评分	1-5 分制，标记问题的紧急程度（5分为最高）
统计输出	情感分布、评分分布、紧急问题 TOP5
可视化	自动生成情感分布饼图和评分分布柱状图

错误分析
误报案例（2条） ：用户提建议（如“价格太贵了，便宜点就好了”），AI 判为负面。原因是模型没有区分“抱怨”和“正式差评”。

改进方向：优化 Prompt，教模型区分“建议类表达”和“明确差评”。

为什么更看重召回率？
在差评预警场景下，漏报的代价比误报更大。漏掉一条真实的差评，可能意味着错过一个严重 Bug（如闪退、扣费纠纷），导致用户流失。而误报只是多花几分钟人工复核。

⚙️ 使用方法
1. 获取 DeepSeek API Key
访问 https://platform.deepseek.com/

注册账号 → API Keys → 创建新密钥

新用户赠送 500 万 tokens（免费）

2. 配置 API Key
在代码第 10 行填入你的 DeepSeek API Key：

python
DEEPSEEK_API_KEY = "sk-你的真实密钥"
3. 安装依赖
bash
pip install pandas openai matplotlib
4. 运行程序
bash
python analyze_reviews.py
5. 选择 CSV 文件
弹窗选择你的评论 CSV 文件

6. 等待分析完成
每 10 条自动保存进度

结果保存在原文件目录的 _output 文件夹

📤 输出文件
文件	说明
